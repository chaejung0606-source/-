# 오류 해결 가이드

각 항목은 `발생 상황 / 원인 / 사용자 조치 / 관리자 조치 / 개발자 확인 사항` 형식입니다.

---

### 로그인이 되지 않음
- 발생 상황: 학번·비밀번호 입력 후 "로그인 실패".
- 원인: 아이디(학번)·비밀번호 불일치, 미가입, 세션 문제.
- 사용자 조치: 학번·비밀번호 재확인. 회원가입 여부 확인.
- 관리자 조치: 신청자 정보 → [비밀번호 재설정]로 새 비밀번호 발급.
- 개발자 확인: Supabase auth 상태, `lib/auth.ts` login 흐름.

### 비밀번호를 잊어버림
- 발생 상황: 비밀번호 기억 안 남.
- 원인: 분실. 셀프 찾기 기능 없음.
- 사용자 조치: 사업단에 문의.
- 관리자 조치: [비밀번호 재설정](8자↑)로 발급 후 안내.
- 개발자 확인: `/api/admin/reset-password`.

### 회원가입이 되지 않음
- 발생 상황: [회원가입 후 시작하기] 눌러도 진행 안 됨.
- 원인: 필수 항목 누락, 비밀번호 8자 미만/불일치, 미동의, 학번 중복.
- 사용자 조치: 빨간 `*` 항목·비밀번호·동의 확인. 이미 가입이면 로그인.
- 관리자 조치: 필요 시 [신청자 등록]로 대리 생성.
- 개발자 확인: `/api/auth/register`.

### 지원금 유형/프로그램이 조회되지 않음
- 발생 상황: 홈/신청에서 유형·프로그램이 안 보임.
- 원인: 신청 기간 아님, 프로그램 비활성, 자격 대상 아님.
- 사용자 조치: 신청 기간·대상 확인, 사업단 공지 확인.
- 관리자 조치: [신청폼 편집]에서 프로그램 활성·기간 설정.
- 개발자 확인: `type-periods`, `programs`, audience 판정.

### 대여/신청 버튼이 보이지 않음
- 발생 상황: 신청 진행 버튼이 없음.
- 원인: 로그인 필요, 기간 아님, 지원신청 승인 전(연계 유형).
- 사용자 조치: 로그인, 기간 확인, 지원신청 승인 여부 확인.
- 관리자 조치: 지원신청 승인 처리 또는 기간 조정.
- 개발자 확인: `app/apply/page.tsx` 게이트 조건.

### 이미 예약된 공간·시간을 신청함
- 발생 상황: "이미 신청·예약된 장소·시간대입니다".
- 원인: 같은 장소·시간대 중복.
- 사용자 조치: 예약 캘린더 확인 후 다른 시간 선택.
- 관리자 조치: 필요 시 상세에서 조정.
- 개발자 확인: `app/api/space-rental/route.ts` 겹침 검사(sameSpace/overlaps).

### 대여기간/시간 입력 오류
- 발생 상황: "종료 일시가 시작 일시보다 늦어야 합니다".
- 원인: 종료<시작(같은 날), 또는 종료일이 시작일보다 빠름.
- 사용자 조치: 날짜·시간 재확인. 여러 날이면 종료일 지정.
- 관리자 조치: 직접 신청/수정 시 동일 규칙.
- 개발자 확인: slotInt/endDate 비교 로직.

### 개인정보 동의 체크 누락
- 발생 상황: "동의 항목에 모두 체크해주세요".
- 원인: 필수 동의 미체크.
- 사용자 조치: 모든 필수 동의 체크.
- 관리자 조치: 해당 없음.
- 개발자 확인: ConsentChecklist 검증.

### 서명 입력 오류
- 발생 상황: 서명이 저장되지 않음.
- 원인: 캔버스 미지원/터치 문제.
- 사용자 조치: 새로고침, 또는 서명 이미지 업로드로 대체.
- 관리자 조치: 해당 없음.
- 개발자 확인: SignaturePad, 이미지 dataURL.

### 신청 완료(제출)가 되지 않음
- 발생 상황: [신청 제출] 눌러도 안 넘어감.
- 원인: 필수 항목 미작성, 글자수 미달/초과, 네트워크.
- 사용자 조치: 안내창의 미작성 항목 보완 후 재제출.
- 관리자 조치: 보완요청으로 안내.
- 개발자 확인: validateStep, applications insert.

### 파일 업로드 실패
- 발생 상황: 업로드가 안 됨/오류.
- 원인: 형식(PDF·이미지 외), 용량 초과, 세션 만료.
- 사용자 조치: 형식 변환, 재로그인 후 재시도.
- 관리자 조치: 해당 없음.
- 개발자 확인: storage 버킷 정책, `lib/upload.ts`.

### 승인 상태가 변경되지 않음(관리자)
- 발생 상황: 상태 저장 후 반영 안 됨.
- 원인: 저장 실패, 캐시.
- 사용자 조치: 마이페이지 새로고침.
- 관리자 조치: 신청 상세 [저장] 재시도, 상태 옵션 확인.
- 개발자 확인: `PATCH /api/applications/[id]`.

### 공간대여 승인 시 캘린더 반영 실패(HTTP 오류)
- 발생 상황: "승인되었으나 구글 캘린더 반영에 실패했습니다".
- 원인: 웹훅 URL 오류, Apps Script 배포/권한, 리다이렉트 처리.
- 사용자 조치: 해당 없음.
- 관리자 조치: [대여가능공간] → [연결 테스트]. 실패 시 Apps Script를 최신 코드·액세스 '모든 사용자'로 재배포.
- 개발자 확인: `callWebhook`(302 GET 팔로우), `docs/space-rental-apps-script.gs`.

### 이용결과 조회가 안 됨
- 발생 상황: 연락처 입력해도 신청 건 조회 안 됨.
- 원인: 신청 시 연락처와 불일치, bookingRole 미지정 폼.
- 사용자 조치: 신청 때 입력한 번호로 재조회.
- 관리자 조치: 신청폼 연락처 항목에 예약 연결 '연락처' 지정.
- 개발자 확인: `requestPhones`/`phoneMatches`.

### 관리자 화면 접근이 되지 않음
- 발생 상황: 특정 관리자 메뉴가 안 보이거나 401.
- 원인: 프로그램 관리자 메뉴 권한 미부여, 세션 만료.
- 사용자 조치: 해당 없음.
- 관리자 조치: 관리자 설정에서 메뉴 권한 부여. 재로그인.
- 개발자 확인: `requireMenu`, AdminLayout NAV 필터.

### 엑셀 다운로드 실패
- 발생 상황: 엑셀이 받아지지 않음.
- 원인: 팝업/다운로드 차단, 대상 미선택.
- 사용자 조치: 해당 없음.
- 관리자 조치: 브라우저 팝업 허용, 항목 선택 후 재시도.
- 개발자 확인: `lib/excel-export`.

### 권한 오류(401/403)
- 발생 상황: API 요청이 Unauthorized.
- 원인: 세션 만료, 권한 부족.
- 사용자 조치: 재로그인.
- 관리자 조치: 역할·메뉴 권한 확인.
- 개발자 확인: `getAdminSession`, `requireMenu`, `requireExpense`.

### 서버 오류(500)
- 발생 상황: 저장/조회 중 서버 오류.
- 원인: Supabase 연결, 컬럼 누락, app_config 오류.
- 사용자 조치: 잠시 후 재시도.
- 관리자 조치: 상태 확인 후 재시도, 지속 시 개발자에 연락.
- 개발자 확인: 서버 로그, `withMissingColumnRetry`, 환경변수.

### 네트워크 오류
- 발생 상황: 요청이 실패/지연.
- 원인: 인터넷 연결 불안정.
- 사용자 조치: 연결 확인 후 재시도.
- 관리자 조치: 해당 없음.
- 개발자 확인: 재시도 로직.

### 모바일 화면 오류
- 발생 상황: 모바일에서 레이아웃/버튼 문제.
- 원인: 오래된 브라우저/캐시.
- 사용자 조치: 최신 브라우저, 캐시 삭제, 세로/가로 전환.
- 관리자 조치: 해당 없음.
- 개발자 확인: 반응형 클래스.

### 브라우저 호환성 문제
- 발생 상황: 특정 브라우저에서 오작동.
- 원인: 구형 브라우저(IE 등) 미지원.
- 사용자 조치: 크롬·엣지·사파리 최신 버전 사용.
- 관리자 조치: 해당 없음.
- 개발자 확인: 폴리필/지원 범위.

### AI 초안 오류(관리자)
- 발생 상황: 대리신청 서술 초안 버튼 오류/미동작.
- 원인: AI 키 미설정.
- 사용자 조치: 해당 없음.
- 관리자 조치: 관리자 설정 → AI 초안 작성 키 저장(또는 환경변수 ANTHROPIC_API_KEY).
- 개발자 확인: `lib/ai-config`, `/api/ai/draft`.
