# 테스트 체크리스트

## ⚠️ 테스트 환경에 대한 정직한 고지

이 문서를 작성한 **개발 샌드박스에는 Supabase 백엔드 자격증명·테스트 계정·한글 폰트가 없어, 로그인 이후(사용자/관리자) 실제 클릭 테스트를 수행할 수 없었습니다.** (앱은 Supabase 연결이 없으면 초기화 단계에서 예외가 발생합니다.)

따라서 결과는 다음 기준으로 정직하게 표기합니다.

- **코드검증**: 코드(라우트·컴포넌트·검증 로직·문구)를 정독하여 동작·경로·입력·오류 문구를 확인함. (일부는 타입체크 통과 확인)
- **런타임 미실행**: 실제 화면에서 클릭·입력하는 런타임 테스트는 수행하지 못함 → 백엔드+테스트 계정 환경에서 재테스트 필요.

> 요청서 10항목의 "미검증 표시 금지, 실제 테스트 후 작성"은 백엔드가 연결된 환경을 전제로 합니다. 이 저장소 환경에서는 그 조건이 충족되지 않아, **거짓으로 '검증 완료'라고 표기하지 않고** 위와 같이 사실대로 구분했습니다. 아래 "런타임 재테스트 절차"대로 실행하면 실제 검증으로 갱신할 수 있습니다.

## 런타임 재테스트 절차

1. `.env.local` 에 Supabase 자격증명 설정(`.env.local.example` 참고).
2. `npm run dev` (또는 배포 URL) 로 앱 실행.
3. 테스트 사용자/관리자 계정 준비.
4. 아래 표의 각 기능을 실제로 수행하고 결과를 갱신.
5. `node scripts/capture-docs-screenshots.js` 로 화면 캡처 생성.
