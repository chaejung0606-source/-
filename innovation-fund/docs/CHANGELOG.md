## v1.1.203 / 2026-07-14

### 기능 변경
- **신청 목록 — 하위 프로그램 드롭다운 필터**: 필터 영역에 '프로그램' 드롭다운 추가(현재 표시 대상 신청들의 프로그램명 목록). 선택 시 해당 프로그램 신청만 표시, 다른 필터·저장(sessionStorage)과 연동.
- **지원신청 금액 질문 자동 산정(오류 수정)**: 관리자가 비용 블록 대신 **일반 숫자 질문**(라벨에 '(원'·'금액'·'~비')으로 금액을 받는 폼에서 합산이 되지 않던 문제 수정 — 근무·비용 블록 + 금액 질문을 모두 합산해 지원신청·지원금신청 동일하게 신청 금액 자동 산정.
- **신청 상세 — 중복 제거·누락 방지 보강**: 상단 '프로그램명'과 동일한 항목(폼 질문·상세 행)은 중복 표시하지 않음. 신청폼 스키마를 불러오지 못하는 경우(구버전·프로그램 삭제)에도 답변 순서 표시 + 특수 블록(행사 장소·비용·근무기록·구성원)을 빠짐없이 표시.

## v1.1.202 / 2026-07-14

### 기능 변경
- **신청 목록 — 하위 프로그램명 표시**: 신청 유형 칸에 해당 신청의 프로그램명(하위 프로그램)을 함께 표시해 확인·검색이 가능하도록 함. (검색은 이름·학번·프로그램명·역할 매칭, `progNameOf`에 스키마 폼·소학회 프로그램명 보강)
- **신청 상세 — 지원신청(활동 전) '관리자 확인 입력' 숨김**: 지급이 없는 지원신청 단계에서는 계좌·주민번호 확인 입력 섹션을 표시하지 않음.
- **신청 상세 — 구성 정리**: 스키마 폼 신청은 '신청 상세 내용(비용·자동 산출)' 별도 섹션을 제거(신청폼 순서 표시로 충분). 프로그램명(하위 프로그램)을 기본 정보 위로 이동. 하단 '신청 금액·자동 산정 금액'은 기존 위치 유지.
- **지원신청 금액 자동 산정**: 지원신청(활동 전)에서도 근무·비용 항목을 작성하면 신청 금액이 자동 계산·표시되도록 변경(항목이 없으면 0).
- **신청서 금액 천단위 쉼표**: 신청자가 작성한 신청서(금액/number 항목)를 천단위 콤마로 표시.

## v1.1.201 / 2026-07-14

### 버그 수정
- **접수번호 중복으로 신청 저장 실패 수정 (`duplicate key value violates unique constraint "applications_receipt_number_key"`)**: 접수번호 생성 트리거가 `COUNT(*)+1`을 써서 ① 취소·탈퇴·테스트 삭제로 행이 빠지면 이미 쓰인 번호를 재생성하고 ② 동시 제출 시 같은 번호를 만들어 UNIQUE 제약을 위반하던 문제.
  - **DB 수정(필수, 1회 실행)**: `supabase/fix-receipt-number.sql`을 Supabase SQL Editor에서 실행. '그 해 최대 번호 + 1' 방식 + 연도별 자문 잠금(`pg_advisory_xact_lock`)으로 삭제·동시성 모두 대응, `SECURITY DEFINER`로 RLS 영향 제거. (신규 설치용 `supabase/schema.sql`도 동일하게 갱신)
  - **앱 방어 보완**: 신청 제출 시 접수번호 중복 오류가 나면 새 번호로 자동 재시도(`insertApplicationWithReceiptRetry`) — 드문 동시 제출 경쟁을 사용자 오류 없이 흡수.
