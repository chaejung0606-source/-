## v1.1.191 / 2026-07-09

### 기능 변경 (개선 필요사항·기능 누락 항목 해결)
- **상태 변경 이력(감사 로그)**: 관리자가 검토/지급 상태를 바꿀 때마다 일시·관리자·역할·이전→이후·메모를 신청 데이터(status_history)에 누적 기록하고, 신청 상세에 타임라인으로 표시
- **반려·보완 사유 필수화**: 검토 상태를 반려·보완요청으로 저장할 때 안내 메모(사유)가 비어 있으면 화면과 서버 양쪽에서 저장을 차단
- **동일 유형 중복 신청 방지**: 신청 제출 직전 같은 종류의 유효 신청(취소·반려 제외)이 있으면 확인창으로 경고 — 반려·취소 후 재신청은 정상 허용
- **비밀번호 기준 통일**: 마이페이지 비밀번호 변경도 회원가입과 동일하게 8자 이상으로 통일(기존 6자)
- QA 런타임 클릭 테스트 27→28개로 확장(사유 필수화 차단·감사 로그 기록 검증 추가), 전부 통과
- 매뉴얼 웹북 15·16장에 위 항목을 '해결'로 반영

### 참고
- `applications.status_history` 컬럼이 추가됩니다(`ADD COLUMN IF NOT EXISTS`). 배포 DB에 미반영 시에도 앱은 정상 동작하며 이력만 저장되지 않습니다 — 운영 DB에 마이그레이션 적용을 권장합니다.

- **이용안내(/guide) = 관리자 편집 내용 그대로**: 자동 생성 섹션 제거로 관리자 화면과 신청자 화면 항상 일치. 기본 안내를 **신청자 매뉴얼(12개 섹션·표 포함)** 로 전면 교체
- 이용안내 **사이드바 링크: 새 탭**으로 열림(작은 창은 '사이드바 링크' 탭 옵션으로 선택 가능)
- 이용안내 버그 수정: 링크 추가 후 사이트설정 상태 새로고침(저장 시 링크 삭제 방지) + /guide 캐시 금지(저장 직후 옛 내용 방지)
- **관리자 비밀번호 게이트: 메뉴 클릭 시에만 확인** — 메뉴 내부 이동(목록↔상세)은 재확인 없음(lib/pw-gate, 탭 단위)
- 관리자 신청 상세: **신청자가 작성한 신청서 한 카드 통합**(관리자 확인 입력 최상단 → 기본정보·신청내용·상세·계좌·첨부 폼 그대로, 보기 전용, 파일 클릭 미리보기)
- `supabase/fix-receipt-number.sql`: 접수번호 생성 COUNT+1→MAX+1 (삭제 후 번호 충돌·신청 실패 방지 — Supabase SQL Editor에서 별도 실행)

### 매뉴얼 변경
- `docs/applicant-manual.md`(신청자 이용 매뉴얼 초안)·`feature-manual-map.md`(기능-매뉴얼 매핑표)·`manual-auto-update-plan.md`(자동 반영 개발안) 추가
