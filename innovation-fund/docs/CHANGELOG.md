## v1.1.184 (QA 수정) / 2026-07-08

### 버그 수정
- **날짜 판정 KST 통일**: UTC 기준 '오늘' 계산으로 한국시간 새벽 0~9시에 하루 어긋나던 문제 — 홈 팝업 게시기간·'오늘 하루만 보기', 유형별 신청기간(isTypeOpen), 프로그램 신청기간(isProgramActive), 지원유형 모달 기준일 (lib/kst.ts 신설)
- **관리자 직접 등록 사진 삭제**: 이용결과에 비고만 입력해도 신청자 제출 사진이 삭제되던 문제 — 입력 항목만 전송·병합 저장
- **반려 건 캘린더 점유**: 승인→반려 시 구글 이벤트가 남아 보이지 않게 시간을 점유하던 문제 — 반려 시 이벤트 삭제 + 잔여 이벤트는 외부 일정으로 표시
- **반복 대여 상한 불일치**: 플랫폼 회차 전개 상한(매주 60회)과 구글 무제한 반복의 차이로 장기 반복 충돌검사 누락 가능 — 상한 2년 커버 상향 + 반복 종료일 시작일+2년 제한

## v1.1.183 / 2026-07-07

### 기능 변경 (공간대여)
- **접수번호 자동 부여**(SR-YYYY-NNN): 신규 신청 접수 시 + 기존 건 접수순 소급 부여. 신청목록·이용결과·신청자 목록·사진 PDF에 표시
- **신청목록 간단히 보기**: 카드 기본 접힘 — 접수번호·상태·요약 제목(공간·일시·신청자), 파악 불가 건은 접수번호만. 클릭해 펼쳐 상세 확인
- **검색**: 접수번호·대여공간 텍스트 + 일자(반복 회차 포함) — 신청목록·이용결과 제출내역 공통
- **반복 대여**: 직접 신청·대여 일정 입력에 매주/매월 + 반복 종료일. 플랫폼 캘린더 회차별 표시, 구글 캘린더 반복 일정(EventSeries) 등록(Apps Script 갱신 필요), 충돌검사 전체 회차 대상
- **일정 겹침 방지**: 관리자 일정 저장 시 다른 신청·구글 예약과 겹치면 경고(확인 시에만 강제 저장). 이용결과 제출 페이지에 접수번호 표기
- **버그 수정**: 일정 수정 시 새 캘린더 이벤트 ID 미저장으로 구글 이벤트가 중복 생성되던 문제, 플랫폼 캘린더에 접수건+자체 생성 구글 이벤트가 이중 표시되던 문제, 웹훅 연결 테스트 후 [저장] 누락 안내 추가, 캘린더 이벤트 제목 [공간대여] 접두어 제거, 캘린더에 사용 목적 표시, 이용결과 서류 삭제, 관리자 직접 등록이 신청자 제출 명단·답변을 지우던 문제(병합 저장)

## v1.1.182 / 2026-07-07

### 기능 변경 (공간대여 전면 개편)
- **서류 제출형 신청**: 신청폼 편집의 '예약 연결' 제거 — 신청자는 폼으로 서류만 제출(파일 업로드 항목 지원: HWP·PDF·오피스·이미지, 끌어다 놓기·여러 개), 대여 일정은 관리자가 신청목록에서 직접 입력해 캘린더 반영
- **신청목록 인라인 심사**: 모달 없이 카드에서 제출 서류 다운로드·미리보기(이동형 창), 대여 일정 입력(공간·기간·시간·종일), 메모 + 승인/보완요청/반려/삭제
- **이용결과 제출 독립 페이지**(/space-rental/result): 연락처 조회 삭제 → 전체 신청목록에서 건별 [이용결과 제출] → 해당 건 폼. 관리자 이용결과 폼이 있으면 그 폼만 표시
- **이용결과 제출내역**: 승인된 건 전체 표시(미제출 포함), 관리자 직접 등록(명단·사진·비고), 신청자 목록 숨김 토글, **이용 사진 PDF**(플랫폼 공통 인쇄 양식)
- '대여 가능한 장소' 아래 **신청서류 다운로드** 박스(공간 대여 신청서 학생용/부서·외부용, 공간대여 명단 + 제출서류 안내)
