## 1. 실측 환경

- **도달성(curl)**: 배포 주소·일반 웹 모두 200. → 정책 차단 아님.
- **로컬 실행**: `.env.local`에 더미 Supabase URL(`localhost:54321`)·`NEXT_PUBLIC_USE_MOCK=true`로 `next dev` 기동. 공개 API 라우트는 백엔드 미도달 시 내장 기본값으로 폴백하여 **공개 화면은 정상 렌더**.
- **실 브라우저 클릭**: 로컬 앱을 루프백(프록시 미경유)으로 Chromium 자동화 → 공개 화면 클릭·게이트·모바일 실측. (`scripts/local-public-test.mjs`, 재현 가능)
- **배포 사이트 브라우저 자동화는 불가**: 이 환경의 필수 송신 프록시가 curl의 CONNECT는 터널링하지만 Chromium의 TLS 핸드셰이크는 `ERR_CONNECTION_RESET`으로 리셋 → **사이트는 도달하나(=curl 200) 배포 사이트 브라우저 자동화는 프록시 한계로 불가**. 정책 차단과는 다른 원인.

## 4. 코드 감사 (현재 트리 정독 — 이전 보고서 §4 재검증 결과 정확)

### 4-1. 제출 경로 — 견고 (통과)
`components/apply/ApplyForm.tsx` 제출 핸들러 실측 확인:
- **중복제출 방지**: `submitting` state로 버튼 `disabled`(`ApplyForm.tsx:823,826`).
- **멈춤(stuck) 없음**: 제출 로직이 `setSubmitting(true)` → `try { … } finally { setSubmitting(false) }` 구조(`:578,626`)로 모든 조기 반환/오류 경로에서 상태 복구.
- **접수번호 동시성 방어(이중화)**: 클라이언트 `insertApplicationWithReceiptRetry`(`lib/app-mapper.ts:10`)가 접수번호/UNIQUE 충돌 시 최대 5회 백오프 재시도 + DB 트리거 `generate_receipt_number()`가 `pg_advisory_xact_lock` + `MAX+1`(`supabase/schema.sql:132-138`)로 직렬화.
- **미로그인 제출 차단**: `supabase.auth.getUser()` 재확인 후 없으면 `/login`으로 유도.

### 4-2. 검증 로직 — 광범위 (통과)
유형별 필수값 검증이 `reasons.push(...)`로 촘촘히 누적되며, 동의·서명·계좌(fund)는 버튼 `disabled` 조건에도 포함.

### 4-3. ⚠️ 개선 권고 — 파일 업로드 클라이언트 용량 가드 부재 (버그 아님)
`lib/upload.ts`·`components/apply/DocumentSlotsSection.tsx`·`FileUploadSection.tsx`: 확장자/타입(`isAllowedDoc`) 검증은 있으나 **파일 크기 상한 검사 없음**(`file.size`는 표시용으로만 사용). 대용량 업로드 시 친절한 "용량 초과" 안내 없이 Storage 원문 오류가 노출될 수 있음. 서버측(Storage 버킷) 제한 설정에 의존 → 이 환경에서 실동작 확인 불가.
- 권고: 업로드 전 `file.size` 클라이언트 검사 + 안내문(예: 10MB). **요구사항 변경이므로 임의 반영하지 않고 권고로만 보고.**

### 4-4. 사소한 불일치 (영향 낮음)
`lib/upload.ts`의 `isAllowedDoc`는 확장자 `gif`/`heif`를 허용하나, 파일선택 필터 문자열 `ACCEPT_DOC`(`.pdf,.jpg,.jpeg,.png,.webp,.heic`)에는 `gif`·`heif` 미포함 → 선택창 필터와 실제 허용범위 불일치. 기능 오류 아님.

### 4-5. 코드 감사만으로 판정 불가 (실 백엔드 필요)
사전→지원금 연계 정합성, RLS 기반 타인 신청/파일 접근차단, 신청기간 서버측 차단, 소학회 연결ID, 공간대여 시간겹침(DB+캘린더), 실 알림/Drive/캘린더 연동.

## 7. 최종 판정

- 이전 보고서의 대전제("네트워크 차단으로 실측 불가")는 **성립하지 않음** — 배포 사이트는 이 환경에서 도달 가능(curl 200).
- **이 환경에서 검증 가능한 범위**(공개 화면 실브라우저 클릭 7/7, 제출/검증/중복방지 코드 감사)는 **모두 정상**. 앱 JS 오류 0건.
- 다만 **"신청자 기능 운영 가능"을 인증(certify)하려면** 인증·제출·프로덕션 DB 저장값 검증이 필요하며, 이는 **실 Supabase 자격증명 + 테스트 학생 계정**이 있어야 가능. 운영 DB 오염을 피하려면 별도 스테이징 백엔드/계정 권장.
- 즉시 조치 권고: **파일 업로드 클라이언트 용량 가드 추가**(§4-3), `ACCEPT_DOC`/`isAllowedDoc` 확장자 목록 일치(§4-4).
