## 0. 이 매뉴얼을 읽는 법 (3가지 핵심만 먼저)

신청에서 가장 많이 막히는 3가지입니다. 이것만 기억해도 절반은 끝납니다.

1. **신청은 “두 종류”가 있습니다.**
   - **지원신청(활동 전)** = 활동을 “하기 전에” 참여를 신청 → 사업단 승인
   - **지원금 신청(활동 후)** = 활동을 “마친 뒤” 돈(지원금)을 신청
   - 둘은 다른 버튼입니다. 지금 내가 하려는 게 무엇인지 먼저 정하세요.
2. **돈은 “연구통합관리시스템에 등록한 본인 명의 계좌”로만 들어옵니다.**
   - 플랫폼에 입력한 계좌 + 통장 사본 + 연구통합관리시스템 계좌, **세 곳의 예금주·계좌번호가 모두 같아야** 지급됩니다.
   - 본인 명의가 아니면 지급이 **보류**됩니다.
3. **빨간 글씨 경고/알림창은 “지금 멈추고 고치라”는 신호입니다.**
   - 알림창(팝업) 메시지는 대부분 “어디를 어떻게 고치라”까지 알려줍니다. 닫지 말고 그대로 따라 하세요.

## 1. 신청 프로세스 분석 — 전체 단계 정의

신청자의 여정을 9단계로 정의합니다.

| 단계 | 이름 | 신청자가 하는 일 | 끝났다는 신호 |
|---|---|---|---|
| 0 | 자격 확인 | 내가 지급 대상인지 확인(재학생/대학원생/수료 2년 이내 등) | — |
| 1 | 회원가입 | 학번으로 계정 생성 + 본인 계좌 입력 | 로그인 화면 진입 |
| 2 | 로그인 | 학번·비밀번호 입력 | 홈 화면 진입 |
| 3 | 종류 선택 | 지원신청/지원금 신청 중 선택 → 세부 종류 선택 | 신청서 첫 화면 |
| 4 | 자격 게이트 통과 | (해당 시) 신청기간·신청대상 조건 통과 | 신청서 1단계 표시 |
| 5 | 신청서 작성 | 4단계(기본정보→신청내용→서류→동의·서명) 작성 | “신청 제출” 버튼 활성 |
| 6 | 제출 | 최종 동의·서명 후 제출 | “정상 접수” 완료 화면 + 접수번호 |
| 7 | 검토 추적 | 마이페이지에서 검토/지급 상태 확인 | 상태가 “승인/지출완료”로 변경 |
| 8 | 보완·지급 | (필요 시) 보완요청 대응 → 본인 계좌로 지급 | 지급 상태 “지출완료” |

> **지원신청 → 지원금 신청 연계**: 지원신청을 먼저 한 경우, 지원금 신청 시 그 내역을 고르면 중복 항목이 자동 입력됩니다. (지원신청을 꼭 먼저 해야만 지원금 신청이 되는 것은 아니며, 신청 자격은 프로그램별 “신청대상” 설정으로 판정됩니다.)

## 3. 정상 흐름 vs 예외 흐름 설계

### 3.1 정상 흐름 (Happy Path)

```
회원가입 → 로그인 → [홈] 종류 선택
  → (프로그램형이면) 프로그램 선택
  → 1단계 기본정보·동의 → 2단계 신청내용 → 3단계 서류 → 4단계 동의·서명
  → 신청 제출 → "정상 접수" + 접수번호
  → 마이페이지에서 상태 추적 → 승인/지출완료
```

### 3.2 예외 흐름 (Branch & Recovery)

- **자격 게이트에서 차단** → 안내 화면의 “뒤로 가기” → 다른 종류 선택 또는 사업단 문의
- **기간 외(성과형)** → 신청기간 badge 확인 → 기간 내 재진입
- **작성 중 중단** → “임시저장” → 마이페이지 “이어서 신청”
- **제출 후 오타 발견** → 마이페이지에서 “신청 취소”(되돌릴 수 없음) → 다시 신청
- **보완요청 수신** → 마이페이지 “수정 후 재제출”
- **지원신청 반려** → 지원금 신청 불가, 사유 확인 후 사업단 문의
- **지급 보류** → 계좌(본인 명의·연구통합관리시스템 등록) 점검

## 5. 화면별 상세 가이드

각 화면은 **[목적] / [입력 방법] / [주의] / [예시] / [자주 나는 오류]** 순으로 정리합니다.

### 5-1. 회원가입

**[목적]** 학번 기반 계정을 만들고, 지원금 받을 본인 계좌를 등록합니다.

**[입력 방법]**
1. 로그인 화면에서 **“회원가입”** 탭을 누릅니다.
2. **학적 유형**을 먼저 선택: **대학생 / 대학원생**. (선택해야 아래 항목이 나타남)
3. 공통 입력: 학번(숫자만) · 이름 · 비밀번호(8자 이상) · 비밀번호 확인 · 캠퍼스.
4. **대학생**: 단과대학 → 학과를 선택(검색·직접입력 가능). 목록에 없으면 직접 입력.
5. **대학원생**: 대학원 종류(일반/전문) 선택 후, 단과대학·학과·전공을 직접 입력.
6. 연락처(010-0000-0000) · 이메일(id@kangwon.ac.kr).
7. **계좌 정보**: 은행 선택 + 예금주 + 계좌번호(‘-’ 없이).
8. **개인정보 수집·이용 동의** 체크 → **“회원가입 후 시작하기”**.

**[주의]**
- 계좌는 **본인 명의**여야 합니다. 안내문구: *“본인 명의 계좌만 지급 가능하며, 정보가 다르면 지급이 보류될 수 있습니다.”*
- 비밀번호는 분실 시 사업단이 “재설정”만 가능하고 **원문 조회는 불가**합니다. 꼭 따로 보관하세요.

**[예시]**
- 학번 `20251234`, 연락처 `010-1234-5678`, 이메일 `coss@kangwon.ac.kr`, 계좌 `국민은행 / 홍길동 / 12345601234567`.

**[자주 나는 오류]** → E1~E6 참고.

---

### 5-2. 로그인

**[목적]** 내 계정으로 접속.

**[입력 방법]** 학번 + 비밀번호 → **“로그인”**. (처리 중에는 “처리 중…” 표시)

**[주의]**
- 로그인 아이디는 **학번**입니다(이메일 아님).
- 관리자가 대신 계정을 만들어 준 경우, 안내받은 학번/비밀번호로 동일하게 로그인합니다.

**[자주 나는 오류]**
- *“학번 또는 비밀번호가 올바르지 않습니다.”* → 학번 숫자/비밀번호 대소문자 확인. 계속 실패 시 사업단에 **비밀번호 재설정** 요청.

### 5-3. 홈 — 종류 선택

**[목적]** 지금 할 신청의 “종류”를 고릅니다.

홈에는 두 박스가 있습니다.

- **📝 지원신청 (활동 전)**: 근로장학금 · 프로그램 참여지원비 · 진행요원비
- **💸 지원금 신청 (활동 후)**: 위 3종 + 성적 우수 · 경진대회 입상 · 자격증 취득

**[입력 방법]** 원하는 카드의 **“지원신청하기 / 지원금 신청하기”** 버튼을 누릅니다.

**[주의 — 카드의 안내 문구 읽는 법]**
- 프로그램형 카드의 **“신청 가능 분야”**: 지금 신청 가능한 프로그램명이 보입니다. 없으면 **“현재 신청 가능한 프로그램이 없습니다.”**
- 성과형 카드의 **🗓️ badge**:
  - `상시 신청 가능` = 아무 때나
  - `신청기간 2026-06-01~06-30 · 신청 가능` = 기간 내라 지금 됨
  - `… · 신청 불가` = 기간 밖이라 지금은 안 됨
- 더 자세한 금액·기준은 하단 **“유형별 지급 기준”** 카드를 눌러 확인.

**[예시]** “마이크로디그리(30만원)·부전공(100만원)·복수전공(150만원), 평점 평균 3.0 이상” 등은 유형별 지급 기준에서 확인.

**[자주 나는 오류]** → E11(프로그램 없음).

### 5-4. 자격 게이트 (신청기간·신청대상)

신청서로 들어갈 때 조건에 막히면 아래 화면이 뜹니다. **이건 오류가 아니라 “자격/시기” 안내**입니다.

- **🗓️ 현재 신청 기간이 아닙니다** (성과형) → 표시된 “신청 가능 기간”에 다시 오세요.
- **🔒 미래융합가상학과 학생만 신청할 수 있습니다** → 가상학과 재학생 명단에 있어야 합니다.
- **🔒 지정된 학생만 신청할 수 있습니다** → 사업단이 지정한 학생만 가능.

**[내가 할 일]** “뒤로 가기” → 다른 종류 선택. 본인이 대상인데도 막히면 **사업단에 문의**(명단/지정 누락일 수 있음). → E8·E9·E10.

### 5-5. 신청서 1단계 — 기본 정보 & 동의

**[목적]** 신청자 신원·연락처·(지원금이면)계좌를 확정하고 필수 동의를 받습니다.

**[입력 방법]**
- 로그인 정보가 **자동 입력**됩니다. 학번은 보통 수정 불가(회색).
- 지원금 신청이면 **본인 명의 계좌**(은행·계좌번호·예금주)도 여기서 확인합니다.
- **개인정보 수집·이용 및 신청 동의** 3개 항목에 모두 체크.

**[주의]**
- **예금주 ≠ 이름**이면 경고가 뜹니다: *“예금주(○○)와 이름(○○)이 다릅니다. 본인 명의 계좌인지 확인해주세요.”* → 본인 명의가 맞는지 반드시 확인.
- 동의는 **전부 필수**입니다. 하나라도 빠지면 다음 단계로 못 갑니다.

**[예시]** 동의 3종: ①수집·이용 ②허위 시 취소·환수 ③본인 명의 계좌 확인.

**[자주 나는 오류]** → E5·E7·E32.

---

### 5-6. 신청서 2단계 — 신청 내용 (유형별)

**[목적]** 종류에 맞는 활동/성과 정보를 입력합니다. 입력 항목은 종류마다 다릅니다.

**[유형별 핵심 입력]**
- **프로그램 참여지원비**: 프로그램 선택, 참여기간, 참여내용, 지도교수. (지원금이면 등록비/교통비/숙박비 비용 입력)
- **진행요원비/근로장학금**: 프로그램·역할, **근무상황부**(날짜·시작·종료 시간), 확인자.
- **성적 우수**: 마이크로디그리(학과·과정·이수교과목) 또는 부전공/복수전공(전공명·평점·이수교과목).
- **경진대회**: 대회명·주제·규모(A/B)·개인/팀·수상등급·수상일. 팀이면 팀원 학번·이름.
- **자격증**: 자격증명·발급기관·취득일(YYYY-MM-DD)·난이도.

**[주의 — 근무상황부 규칙]**
- 종료 시간이 시작보다 늦어야 함. 1일 최대 8시간. 같은 날 합계도 8시간 이내. 최대 시간 상한 초과 불가.
- 근로장학금은 **수업시간과 겹치는 시간**을 근로로 등록할 수 없습니다(마이페이지에서 수강 시간표를 먼저 입력).

**[예시]** 경진대회 팀 신청: 본인 포함 **2명 이상**, 모든 팀원의 학번·이름 필수. 지급은 인원수로 나눠 지급(n분의 1).

**[자주 나는 오류]** → E12~E27.

---

### 5-7. 신청서 3단계 — 서류 업로드

**[목적]** 자격·성과·계좌를 증빙합니다.

**[입력 방법]**
- 성과형(성적/경진/자격증)은 **서류 칸이 정해져** 있습니다(각 칸에 드래그 또는 클릭 업로드).
  - 공통: 재학증명서, 신분증 사본, **통장 사본**(예금주=본인) + 종류별 증빙(성적증명서/수상증빙/자격증 사본).
- 업로드 가능한 형식: **이미지(JPG·PNG·WEBP) 또는 PDF**.

**[주의]**
- **재학증명서**는 *“열람용이 아닌, 학교 직인이 날인된 정식 발급본”* 으로 제출해야 합니다(업로드 시 안내창이 뜸).
- **통장 사본**은 *“예금주가 신청자 본인 명의”* 여야 합니다.
- 필수 칸은 비우면 제출 불가.

**[자주 나는 오류]** → E28~E30.

### 5-8. 신청서 4단계 — 동의·서명·제출

**[목적]** 최종 확인 후 제출.

**[입력 방법]**
- 신청 요약(성명·유형·금액) 확인.
- **신청인 서명**: “직접 서명(그리기)” 또는 “서명 이미지 업로드” 중 하나(필수).
- **“신청 제출”** 클릭 → **“정상 접수”** 화면 + **접수번호** 확인.

**[주의]**
- 제출 전 “임시저장”으로 언제든 중단·재개 가능. **임시저장은 아직 접수 아님**(반드시 제출까지).
- 완료 화면 경고: *“허위 신청 또는 부적격 사유 확인 시 지급이 취소되거나 환수될 수 있습니다.”*

**[자주 나는 오류]** → E31·E33·E34.

### 5-9. 마이페이지 — 내 정보·신청 관리

**[목적]** 내 정보 수정과 신청 추적·관리.

**할 수 있는 일**
- **개인정보 수정**: 이름·소속·연락처·이메일·**본인 계좌**. (수정 전 **본인 확인(비밀번호)** 1회)
- **비밀번호 변경**: 새 비밀번호(6자 이상)+현재 비밀번호. 변경 후 **자동 로그아웃** → 재로그인.
- **학적상태변경**: 대학원 진학 등으로 **학번(로그인 아이디)** 이 바뀔 때. **기존 신청 기록은 모두 유지**. 변경 후 새 학번으로 재로그인.
- **수강 시간표**: 근로장학금 신청자 필수(수업시간 근로 방지).
- **임시저장**: “이어서 신청” 또는 “삭제”.
- **신청 내역**: 검토/지급 상태, 진행 단계, 금액, 안내 메모 확인.
- **신청 취소**: 잘못/중복 신청 취소(되돌릴 수 없음, “취소됨”으로 분리 보관).

**[주의]** 학번은 개인정보 수정에서는 못 바꿉니다(회색). 학번 변경은 **학적상태변경**에서만.

**[자주 나는 오류]** → E35~E41.

## 8. 예외 상황 40선과 해결

> 형식: **증상 → 원인 → 해결**

### 로그인·가입
- **E1. “학번 또는 비밀번호가 올바르지 않습니다.”** → 오타/미가입 → 학번(숫자)·비밀번호 재확인, 안 되면 사업단에 재설정 요청.
- **E2. “다음 항목을 모두 입력해주세요: …”** → 필수 누락 → 나열된 항목을 모두 채우기.
- **E3. “비밀번호는 8자 이상이어야 합니다.”** → 가입 비밀번호 8자↑로.
- **E4. “비밀번호가 일치하지 않습니다.”** → 확인란을 동일하게.
- **E5. “개인정보 수집·이용에 동의해야 회원가입이 가능합니다.”** → 동의 체크.
- **E6. 이미 가입된 학번** → 중복 가입 → 회원가입 대신 **로그인**(비번 분실 시 재설정 요청).

### 자격·시기
- **E7. 예금주 불일치 경고** → 예금주≠본인 이름 → 본인 명의 계좌로 수정(타인 명의 불가).
- **E8. “현재 신청 기간이 아닙니다”** → 성과형 기간 밖 → badge의 “신청 가능 기간”에 재진입.
- **E9. “미래융합가상학과 학생만 신청할 수 있습니다”** → 가상학과 명단 미등록 → 본인이 가상학과면 사업단에 명단 확인 요청.
- **E10. “지정된 학생만 신청할 수 있습니다”** → 지정 대상 아님 → 대상이 맞다면 사업단에 지정 요청.
- **E11. “현재 신청 가능한 프로그램이 없습니다.”** → 열린 프로그램 없음/기간 아님 → 유형별 지급 기준·공지 확인 후 모집 시기에 재시도.

### 신청내용(2단계)
- **E12. “신청 가능한 프로그램을 선택해주세요.”** → 프로그램 미선택 → 드롭다운에서 선택.
- **E13. “근무상황부에 근무 기록을 1건 이상 등록해주세요.”** → 근무 미입력 → 날짜·시간 등록.
- **E14. “종료 시간이 시작 시간보다 늦어야 합니다.”** → 시간 역전 → 시간 수정.
- **E15. “일일 최대 근무시간은 8시간입니다.”** → 1일 8시간 초과 → 분할/축소.
- **E16. “같은 날짜 합계가 일일 최대 8시간을 초과합니다.”** → 같은 날 누적 초과 → 해당 날짜 합계 8시간 이내로.
- **E17. “최대 근무시간(…)을 초과하여 등록할 수 없습니다.”** → 상한 초과 → 남은 시간 내로.
- **E18. 수업시간과 겹쳐 근로 등록 불가** → 시간표 충돌 → 마이페이지에서 시간표 입력/수정 후 겹치지 않는 시간으로.
- **E19. “확인자(지도교수·담당자)를 입력해주세요.”** → 근로 지원금 확인자 누락 → 입력.
- **E20. “활동명을/신청 금액을 입력해주세요.”** → 활동지원비 누락 → 입력.
- **E21. 자격증 “취득일은 YYYY-MM-DD 형식으로”** → 형식 오류 → 예: `2026-03-15`.
- **E22. “마이크로디그리 학과/과정을 선택해주세요.”** → 미선택 → 선택.
- **E23. “이수조건을 충족하지 않아 제출할 수 없습니다.”** → 마이크로디그리 요건 미충족 → 사유 확인, 이수 후 신청.
- **E24. “부전공/복수전공 전공명/평점 평균을 입력해주세요.”** → 누락 → 입력.
- **E25. “지원 자격을 충족하지 않아 제출할 수 없습니다.”** → 평점 3.0 미만/미래융합 미이수/이수학점 부족 등 → 사유 확인.
- **E26. “팀 신청은 본인 포함 2명 이상…”** → 팀원 부족 → 2명 이상 입력.
- **E27. “모든 팀원의 학번과 이름을 입력해주세요.”** → 팀원 정보 누락 → 전원 입력.

### 서류(3단계)
- **E28. “[○○] 서류를 업로드해주세요.”** → 필수 칸 비움 → 해당 칸 업로드.
- **E29. “이미지(JPG·PNG·WEBP) 또는 PDF만 업로드할 수 있습니다.”** → 형식 오류 → 변환 후 재업로드.
- **E30. “○○ 업로드 실패: …”** → 네트워크/용량 → 잠시 후 재시도, 파일 용량 축소.

### 제출·동의·서명
- **E31. “신청인 서명 이미지를 업로드해주세요. (필수)”** → 서명 누락 → 직접 서명 또는 이미지 업로드.
- **E32. “개인정보 수집·이용 및 신청 동의 항목에 모두 체크해주세요.”** → 미동의 → 전체 체크.
- **E33. “임시저장 실패: …” / 저장만 하고 미제출** → 네트워크 또는 미제출 → 재저장, 그리고 **반드시 “신청 제출”** 까지.
- **E34. “관리자 계정은 신청서를 제출할 수 없습니다.”** → 관리자 화면 확인용 → 신청자 계정으로 진행.

### 사후·마이페이지
- **E35. 보완요청 수신** → “✍️ 보완 요청이 있습니다.” → “수정 후 재제출”로 보완하여 다시 제출.
- **E36. 지원신청 반려 → 지원금 신청 불가** → “⛔ 반려” → 사유 보기 후 사업단 문의.
- **E37. 지원신청 미승인 → 지원금 신청 대기** → “⏳ 승인 후 가능” → 승인까지 대기.
- **E38. 신청 취소 후 복구 요청** → 취소는 되돌릴 수 없음 → 새로 신청.
- **E39. 학번 변경 후 로그인 안 됨** → 아이디가 새 학번으로 바뀜 → **새 학번**으로 로그인(기록은 유지).
- **E40. 비밀번호 변경 후 로그아웃됨** → 정상 동작 → 새 비밀번호로 재로그인.
- **E41. 마이페이지 “비밀번호가 올바르지 않습니다.”(본인확인)** → 현재 비밀번호 오류 → 정확히 입력, 모르면 재설정 요청.
- **E42. 지급 보류/지연** → 계좌 불일치·연구통합관리시스템 미등록 → 세 곳 계좌 일치 + 연구통합관리시스템 본인계좌 등록 확인.
- **E43. 휴학·졸업 등으로 지급 제한** → 자격 제한 대상 → 자격 요건 확인(재학/수료 2년 이내 등).

## 9. FAQ

- **Q. 지원신청과 지원금 신청, 둘 다 해야 하나요?**
  A. 활동 “전”에는 지원신청(승인용), 활동 “후”에는 지원금 신청을 합니다. 프로그램에 따라 지원신청 단계가 없을 수도 있습니다. 신청 자격은 프로그램별 “신청대상”으로 정해집니다.
- **Q. 임시저장했는데 접수가 안 됐어요.**
  A. 임시저장은 작성 중 보관일 뿐 접수가 아닙니다. 마이페이지에서 “이어서 신청” → **신청 제출**까지 해야 접수됩니다.
- **Q. 계좌를 바꿔야 해요.**
  A. 마이페이지 “개인정보 수정”에서 본인 계좌를 변경하세요. 동시에 **연구통합관리시스템 계좌**도 본인 명의로 맞춰야 지급됩니다.
- **Q. 가족 계좌로 받아도 되나요?**
  A. 안 됩니다. **본인 명의 계좌만** 지급됩니다.
- **Q. 학번이 바뀌었어요(대학원 진학).**
  A. 마이페이지 “학적상태변경”에서 새 학번으로 변경하세요. **기존 신청 기록은 유지**되고, 이후 새 학번으로 로그인합니다.
- **Q. 비밀번호를 잊었어요.**
  A. 사업단에 재설정을 요청하세요(원문 조회는 불가). 받은 임시 비밀번호로 로그인 후 변경 권장.
- **Q. 신청을 잘못했어요.**
  A. 마이페이지에서 “신청 취소”(되돌릴 수 없음) 후 다시 신청하세요. 이미 지급 진행 건은 취소가 제한될 수 있어 문의가 필요합니다.
- **Q. “심의필요”가 떴어요. 제가 뭘 해야 하나요?**
  A. 별도 요청이 없으면 대기하면 됩니다. 추가 자료 요청(보완요청)이 오면 그때 대응하세요.
- **Q. 파일이 안 올라가요.**
  A. 형식은 이미지(JPG·PNG·WEBP)/PDF만 가능합니다. 용량을 줄이거나 잠시 후 재시도하세요.

## 11. 신청자 관점 자기 검토 & 개선안

작성한 매뉴얼을 “처음 신청하는 학생”의 눈으로 다시 점검하고, 남는 어려움과 개선안을 제시합니다.

| 여전히 어려운 점 | 왜 | 매뉴얼·서비스 개선안 |
|---|---|---|
| 지원신청/지원금 신청 구분 | 이름이 유사, 순서 헷갈림 | 홈 버튼 옆에 “활동 전/활동 후” 한 줄 부제 고정 노출. 본 매뉴얼 §0를 신청 첫 화면 상단 배너로 |
| “신청 가능한 프로그램 없음”의 모호함 | 자격 문제인지 시기 문제인지 불명 | 메시지를 “현재 모집 기간이 아닙니다(다음 모집: …)” 또는 “자격 대상이 아닙니다(문의)”로 분기 안내 |
| 자격 차단(가상학과/지정학생) 후 막막함 | 다음 행동이 불명확 | 차단 화면에 “사업단 문의” 버튼+연락처를 바로 노출(현재는 안내문만) |
| 계좌 3중 일치 요구 | 연구통합관리시스템을 모름 | 제출 직전·완료 화면에 “연구통합관리시스템 본인계좌 등록” 체크 안내+링크 상시 노출 |
| 근무상황부 규칙 다수 | 규칙이 분산 | 근무 입력칸 옆에 “1일 8시간/수업시간 제외” 요약 상시 표기 |
| 임시저장≠접수 | 저장 후 안심하고 이탈 | 임시저장 시 “아직 접수되지 않았습니다” 강조 토스트, 마이페이지 임시저장에 빨강 안내 유지 |
| 상태 용어(심의필요 등) | 다음 행동 불명 | 각 상태 badge에 “지금 할 일” 한 줄 툴팁(본 §6 표를 제품에 반영) |
| 보완요청 놓침 | 알림 부재 | 보완요청 시 이메일/문자 알림(향후), 마이페이지 상단 고정 배너 |

**핵심 개선 우선순위(권장)**
1. 차단/오류 화면에 **“다음 행동 + 문의 버튼”** 항상 포함
2. **연구통합관리시스템 계좌 등록** 안내를 제출 흐름 안에 내장
3. **임시저장↔제출** 차이를 시각적으로 더 강하게 구분
