# 관리자 매뉴얼

혁신인재지원금 + 공간대여 플랫폼 — 관리자 실무 운영 매뉴얼입니다.
각 기능은 `기능 설명 → 권한 → 접근 경로 → 처리 대상 → 단계별 처리 → 입력/수정 항목 → 완료 확인 → 주의사항 → 실무 팁 → 오류 해결` 순입니다.

> ℹ️ **관리자 매뉴얼은 화면 캡처를 넣지 않고 텍스트로 운영합니다.** 관리자 화면에는 실제 학생 개인정보(PII)가 표시되므로, 캡처 이미지는 개인정보가 없는 **신청자(사용자) 매뉴얼에만** 넣습니다. 관리자 화면은 실제 화면을 열어 이 문서의 단계 설명과 함께 보세요.

## 관리자 역할 개요

| 역할 | 권한 | 로그인 |
|---|---|---|
| **지출관리자(expense)** | 전체 권한(모든 메뉴) | 기본 아이디 `20182135` + 설정 비밀번호 |
| **프로그램 관리자(program)** | `신청 목록` + **부여받은 메뉴만** | 관리자 설정에서 생성한 아이디/비밀번호 |

부여 가능 메뉴(GRANTABLE_MENUS): 신청자 정보 / 신청폼 편집 / 공간대여 신청 / 사이트 설정. **관리자 설정은 지출관리자 전용(부여 불가).**

**좌측 메뉴 순서**: 신청 목록 · 신청자 정보 · 신청폼 편집 · 공간대여 신청 · 관리자 설정 · 사이트 설정.

## 1. 관리자 로그인


1. **기능 설명**: 지출관리자·프로그램 관리자가 관리자 시스템에 로그인합니다.
2. **관리자 권한 필요 여부**: 로그인 자체는 계정만 있으면 가능.
3. **접근 경로**: `/admin/login` (신청자 로그인 화면 하단 [관리자 로그인] 링크).
4. **처리 대상**: 관리자 계정.
5. **단계별 처리 방법**
   1. `/admin/login` 접속.
   2. **아이디**(placeholder `관리자 아이디`) 입력.
   3. **비밀번호** 입력.
   4. **[로그인]** 클릭 → 성공 시 홈으로 이동, 좌측 관리자 메뉴 노출.
6. **입력/수정 항목**: 아이디, 비밀번호.
7. **처리 완료 후 확인사항**: 좌측 사이드바에 역할 배지(지출관리자 / 프로그램 관리자 · 이름) 표시.
8. **주의사항**: 지출관리자 비밀번호는 [관리자 설정]에서 설정합니다(미설정 시 로그인 불가).
9. **실무 운영 팁**: 프로그램 관리자 계정은 담당자별로 발급하고, 필요한 메뉴만 부여하세요.
10. **오류 발생 시 해결 방법**: `아이디 또는 비밀번호가 올바르지 않습니다.` → 계정 확인. 로그인 문제 진단은 `/api/admin/login-diag` 참고.

## 2. 신청 목록(대시보드 통합)


1. **기능 설명**: 신청 건을 조회·필터·정렬하고, 통계 대시보드·인계·엑셀/인쇄를 처리합니다. (별도 대시보드 페이지는 이 화면 상단에 통합)
2. **관리자 권한 필요 여부**: 모든 관리자(역할별로 보이는 집합이 다름).
3. **접근 경로**: 좌측 **[신청 목록]** (`/admin/applications`). 진입 시마다 **관리자 비밀번호 재확인**(개인정보 보호).
4. **처리 대상**: 접수된 신청 건.
5. **단계별 처리 방법**
   1. 메뉴 진입 → 확인창에 관리자 비밀번호 입력 후 **[확인]**.
   2. 상단 대시보드에서 **관리자 미확인 신청**, **검토/지급 상태별 카운트**를 확인합니다.
   3. 상단 필터로 좁힙니다: **이름/학번 검색**, **신청 유형/검토 상태/지급 상태** 드롭다운, **신청일 범위**, **역할**.
   4. 탭 **[신청 목록]/[취소 목록]** 으로 전환합니다.
   5. 처리할 건은 **[상세]** 를 클릭해 상세 화면으로 이동합니다.
   6. (내보내기) 체크박스 선택 후 **[지출자료]/[심의요청서]** 인쇄, **[선택 목록 다운로드]** 엑셀. 또는 **[전체 목록 다운로드]**.
6. **입력/수정 항목**: 검색어·필터값, 선택 체크박스.
7. **처리 완료 후 확인사항**: 필터 결과 건수, 미확인 신청 수 변화.
8. **주의사항**: 진입 시 비밀번호는 세션에 저장되지 않아 재진입 때마다 확인합니다. 8건 초과 인쇄 시 확인창이 뜹니다.
9. **실무 운영 팁**: 프로그램 관리자는 담당 프로그램의 검토중 건만 보이며 **[지출관리자에게 보내기]** 로 인계합니다. 지출관리자는 대시보드 **프로그램별 미전달** 로 진행 현황을 파악하세요.
10. **오류 발생 시 해결 방법**: `비밀번호가 올바르지 않습니다.` → 재입력. `검색 결과가 없습니다.` → 필터 초기화. 엑셀이 안 받아지면 팝업 차단 해제.

## 3. 신청 상세 — 승인/반려/보완/지급


1. **기능 설명**: 신청 내용을 확인하고 검토/지급 상태 변경, 최종 승인 금액·안내 메모, 계좌·주민번호 확인 입력, 인쇄를 처리합니다.
2. **관리자 권한 필요 여부**: 관리자(상세 접근은 역할 무관).
3. **접근 경로**: 신청 목록 → **[상세]** (`/admin/applications/{id}`).
4. **처리 대상**: 단일 신청 건.
5. **단계별 처리 방법**
   1. 본문에서 기본정보·계좌·신청 내용·첨부파일을 확인합니다(통장 사본 미리보기).
   2. **관리자 확인 입력** 카드에서 **[수정]** → 통장사본·신분증 기준으로 은행·계좌번호·예금주·주민등록번호를 입력하고 저장(저장 시 마스킹).
   3. 우측 **상태 관리** 카드에서 **검토 상태**(예: 승인/반려/보완요청/심의필요)와 **지급 상태**를 선택합니다.
   4. 필요 시 **최종 승인 금액(원)** 을 입력합니다(미입력 시 자동 산정액).
   5. **신청자 안내 메모** 에 반려/보완 사유 등 신청자에게 노출할 안내를 작성합니다. (메모 예시 클릭으로 빠르게 채움)
   6. **[저장]** 을 클릭합니다.
6. **입력/수정 항목**: 검토 상태, 지급 상태, 최종 승인 금액, 안내 메모, 확인 계좌(은행·계좌번호·예금주), 주민등록번호.
7. **처리 완료 후 확인사항**: `저장되었습니다.` alert. 신청자 마이페이지에 상태·안내 메모 반영.
8. **주의사항**: 안내 메모는 **신청자에게 그대로 노출**됩니다. 주민번호·계좌는 저장 시 마스킹됩니다. 반려/보완 시 반드시 사유를 메모에 남기세요.
9. **실무 운영 팁**: 상태 옵션(라벨·색상)이 부족하면 **[옵션 편집]** 으로 추가·수정 후 **[상태 옵션 저장]**. 인쇄 문서는 상단 **[지출자료]/[심의요청서]** 로.
10. **오류 발생 시 해결 방법**: 저장 실패 시 새로고침 후 재시도. Drive 재동기화 실패는 `GOOGLE_SYNC_WEBHOOK_URL` 설정 확인.

## 4. 문서 인쇄

1. **기능 설명**: 지급신청서·지출자료·심의요청서를 PDF로 인쇄/저장합니다.
2. **권한**: 관리자.
3. **접근 경로**: 신청 상세/목록의 **[지출자료]/[심의요청서]** → 인쇄 화면(`/admin/applications/{id}/print?doc=payment|review`).
4. **처리 대상**: 신청 건.
5. **단계별**: (1) 버튼 클릭 → 인쇄 화면 새 창. (2) **[📄 PDF로 저장 / 인쇄]** 클릭. (3) 인쇄 대화상자에서 "PDF로 저장" 선택. (4) 안내된 파일명으로 저장.
6. **입력/수정 항목**: 없음.
7. **완료 확인**: 파일명 `{접수번호} {문서명}_({이름}_{학번})` 로 저장.
8. **주의사항**: 파일명·보관 경로 형식은 [사이트 설정 → 파일 저장 경로]에서 지정.
9. **실무 팁**: 여러 건은 목록에서 체크 후 일괄 인쇄(8건 초과 시 확인창).
10. **오류 해결**: 팝업 차단 시 새 창 허용.

## 5. 서류 인계

1. **기능 설명**: 프로그램 관리자 ↔ 지출관리자 간 신청 서류를 인계/반송합니다.
2. **권한**: 프로그램 관리자(보내기), 지출관리자(반송).
3. **접근 경로**: 신청 목록/상세.
4. **처리 대상**: 신청 건.
5. **단계별**
   - (프로그램 관리자) 서류 확인 후 **[지출관리자에게 보내기]**(목록 일괄) 또는 상세의 **[서류 확인 완료 — 지출관리자에게 보내기]**. 확인창에서 진행.
   - (지출관리자) 보완 필요 시 **[프로그램 관리자에게 반송(보완 요청)]** → 프롬프트에 보완 내용 입력.
6. **입력/수정 항목**: (반송) 보완 요청 내용.
7. **완료 확인**: `{n}건을 지출관리자에게 전달했습니다.` / `프로그램 관리자에게 반송했습니다.`
8. **주의사항**: 인계 단계(reviewStage)에 따라 각 역할에게 보이는 목록이 달라집니다.
9. **실무 팁**: 반송 시 사유를 구체적으로 적어 재작업을 줄이세요.
10. **오류 해결**: 대상 미선택 시 안내창. 새로고침 후 재시도.

## 6. 신청자 정보


지출관리자 전용(또는 메뉴 권한). 진입 시 비밀번호 확인. 탭: **학생 검색 / 프로그램별 신청 가능 학생 / 가상학과 학생**.

### 6-1. 학생 검색·비밀번호 재설정
1. **기능 설명**: 학생 계정 검색, 비밀번호 재설정, 신청정보 확인.
2. **권한**: 지출관리자/메뉴 권한. 3. **경로**: 신청자 정보 → 학생 검색.
4. **처리 대상**: 가입 학생. 5. **단계별**: (1) 검색어 입력. (2) 관리 열의 **[비밀번호 재설정]** → 프롬프트에 새 비밀번호(8자↑) → 학생에게 안내. (3) **[신청정보]** 로 신청 내역 모달 확인.
6. **입력/수정 항목**: 검색어, 새 비밀번호. 7. **완료 확인**: 재설정 성공 alert.
8. **주의사항**: 비밀번호는 화면에 비공개(`•••••• (비공개)`)로 표시됩니다.
9. **실무 팁**: 여러 명은 학번/이름을 띄어쓰기·쉼표로 구분해 한번에 검색.
10. **오류 해결**: 8자 미만 경고. 재설정 실패 시 재시도.

### 6-2. 신청자 등록(대리 계정 생성)
1. **기능 설명**: 관리자가 학생 계정을 직접 생성. 2. **권한**: 지출관리자/메뉴 권한. 3. **경로**: 학생 검색 → **[신청자 등록]**.
4~5. **단계별**: (1) 모달에서 학번*·이름*·비밀번호*(8자↑)·캠퍼스·학과·연락처·이메일·계좌 입력. (2) **[등록]**.
6. **입력 항목**: 위 항목. 7. **완료 확인**: 등록 성공 alert(학번·비밀번호 안내).
8. **주의**: 학번·이름·비밀번호 필수. 9. **팁**: 오프라인 신청 학생 계정을 미리 만들 때 사용. 10. **오류**: 필수 누락/중복 시 안내.

### 6-3. 대리 신청
1. **기능 설명**: 학생 명의로 프로그램 참여지원비를 대리 작성. 2. **권한**: 관리자. 3. **경로**: 학생 행의 **[지원신청]/[지원금]** (`/apply?adminFor={id}&mode=pre|fund`).
4~5. **단계별**: (1) 버튼 클릭 → 해당 학생 정보로 신청서 진입. (2) 폼 작성 후 제출. (서술 항목은 AI 초안 버튼 사용 가능 — AI 키 설정 시)
6. **입력**: 신청서 항목. 7. **완료 확인**: 대리 신청 접수번호 안내.
8. **주의**: 학생 동의 하에 진행. 9. **팁**: AI 초안 키를 설정하면 서술형 초안 자동 작성. 10. **오류**: 대상 정보 로드 실패 시 재시도.

### 6-4. 지정 프로그램 / 알림 보내기
- **지정 프로그램**: 학생 행 **[지정]** → '지정학생만' 프로그램을 체크 저장(해당 학생만 신청 허용).
- **알림 보내기**: 학생 선택 → **[알림 보내기]** → 제목·내용(1명 선택 시 연결할 신청 건 선택) → **[n명에게 보내기]**. 신청자 마이페이지 '관리자 요청 건'에 표시, 회수(삭제) 가능.

### 6-5. 프로그램별 신청 가능 학생
1. **기능 설명**: 승인 신청자 + 지정 학생을 프로그램별로 모아 조회·엑셀 다운로드.
2. **권한**: 지출관리자/메뉴 권한. 3. **경로**: 신청자 정보 → 프로그램별 신청 가능 학생.
4~5. (1) 프로그램 필터·검색. (2) **[엑셀 다운로드]**. 6~10. 파일명 `프로그램별신청가능학생_{날짜}.xlsx`. 표시할 학생 없으면 안내.

## 7. 미래융합가상학과 학생 명단

1. **기능 설명**: 가상학과 재학생 명단 관리(엑셀 업로드·추가·삭제)와 가상학과 전용 신청 유형 설정.
2. **권한**: 지출관리자/메뉴 권한(`/admin/applicants` 또는 `/admin/virtual-students`). 진입 시 비밀번호 확인.
3. **접근 경로**: 신청자 정보 → **가상학과 학생** 탭.
4. **처리 대상**: 가상학과 학생 명단.
5. **단계별 처리 방법**
   1. **가상학과 학생만 신청 가능한 유형** 을 체크하고 **[저장]**.
   2. **[엑셀 업로드]** 로 명단 파일을 올립니다(확인창: 전체 교체 / 추가·갱신 선택).
   3. 또는 **[학생 추가]** 로 학번*·성명·가상학과·본소속학과를 직접 추가.

6. **입력/수정 항목**: 전용 유형 체크, 엑셀 파일, 학번·성명·가상학과·본소속학과·학년·평점·취득학점.

8. **주의사항**: 엑셀 헤더는 학번/성명/가상학과/본소속학과/학년/평점평균/취득학점 등을 인식합니다. 전체 교체 시 기존 명단이 대체됩니다.
9. **실무 팁**: 자격증 지원금·가상학과 전용 유형은 이 명단 기준으로 자격이 판정됩니다.
10. **오류 해결**: `학번이 있는 데이터를 찾지 못했습니다` → 엑셀 학번 열 확인.

## 8. 신청폼 편집


1. **기능 설명**: 프로그램·신청서 양식(폼빌더)·신청기간·템플릿, 유형별 지급기준·자격증을 편집합니다.
2. **관리자 권한 필요 여부**: 지출관리자/메뉴 권한(`/admin/programs`).
3. **접근 경로**: 좌측 **[신청폼 편집]**. 탭: 프로그램 신청 내용 / 프로그램 검색 / 템플릿 설정 / 성과형 신청기한 / 유형별 지급 기준 / 자격증 목록.
4. **처리 대상**: 프로그램·폼·기간·템플릿.
5. **단계별 처리 방법(프로그램 신청 내용)**
   1. 지원금 종류(근로장학금/프로그램 참여지원비/진행요원비/소학회) 선택.
   2. 하위 **프로그램**을 선택하거나 **[프로그램 추가]**.
   3. 수정할 단계(**지원신청 / 지원금 신청**)를 선택.
   4. **프로그램명·역할**, **신청대상**(가상학과만/지정학생만/누구나), **신청기간**(상시/마감/기간)을 설정.
   5. **신청 폼 편집**(SchemaForm)에서 항목을 추가·수정합니다. 항목 추가 → 종류 선택(한 줄 입력·서술·숫자·날짜·시간·드롭다운·표·파일 업로드·**파일 다운로드**·서약·서명·기본정보·개인정보동의·계좌 등). 드롭다운은 선택지별 조건부 추가질문 설정 가능.
   6. 상단 **[저장]** 으로 반영(프로그램·폼·기간·템플릿 동시 저장).
6. **입력/수정 항목**: 프로그램명·역할, 신청대상, 신청기간, 폼 항목(라벨·필수·옵션·글자수·파일 안내·다운로드 파일 등).
7. **처리 완료 후 확인사항**: `저장됨 ✓`. 신청자 화면(미리보기)과 동일하게 반영.
8. **주의사항**: 프로그램 삭제·항목 변경은 **상단 [저장]** 을 눌러야 최종 반영됩니다. 성과형(성적/경진대회/자격증) 신청기한은 **[성과형 신청기한]** 탭에서 학기별로 설정.
9. **실무 팁**: 자주 쓰는 폼은 **[템플릿으로 저장]** 후 다른 프로그램에서 **불러오기**. 공간대여 신청폼은 [공간대여 신청]에서 편집합니다.
10. **오류 발생 시 해결 방법**: `저장할 폼이 없습니다`/`템플릿 이름을 입력해주세요` 등 안내 확인. 조건부 추가질문이 신청 화면에 안 뜨면 옵션 값과 분기 설정 확인.

## 11. 공간대여 신청 관리


1. **기능 설명**: 공간대여 신청 심사(승인/반려/보완), 대여 공간·캘린더·웹훅 설정, 신청/이용결과 폼 편집, 직접 신청.
2. **관리자 권한 필요 여부**: 지출관리자/메뉴 권한(`/admin/space-rental`).
3. **접근 경로**: 좌측 **[공간대여 신청]**. 탭: 신청목록 / 대여가능공간 / 공간대여 신청폼 / 공간대여 캘린더.
4. **처리 대상**: 공간대여 신청, 대여 공간, 폼.
5. **단계별 처리 방법(심사)**
   1. **[신청목록]** 탭에서 건별 **[상세·심사]** 클릭.
   2. 내용을 확인하고 필요 시 대여 공간·일시·신청자 정보를 수정 후 **[수정 저장 (캘린더·시트 반영)]**.
   3. 보완이 필요하면 **관리자 메모** 에 사유를 적고 **[보완요청]**, 부적합하면 **[반려]**, 승인하려면 **[승인]**.
   4. 승인 시 구글 캘린더·시트에 자동 반영됩니다(웹훅 설정 시).
6. **입력/수정 항목**: 대여 공간·일시·신청자 정보·목적·인원, 관리자 메모, 상태.
7. **처리 완료 후 확인사항**: 승인 alert(`승인되어 구글 캘린더·시트에 반영을 요청했습니다.`), 캘린더 반영.
8. **주의사항**: 웹훅 미설정이면 캘린더 수동 등록이 필요합니다. 반영 실패(HTTP 오류) 시 아래 팁 참고.
9. **실무 운영 팁**
   - **[대여가능공간]** 탭에서 공간(공간명·수용인원·사진 여러 장), **구글 캘린더 ID**, **승인 웹훅 URL** 을 설정하고 **[연결 테스트]** 로 확인(`✅ 웹훅 연결 성공!`).
   - **[공간대여 신청폼]** 탭에서 신청 폼/이용결과 폼을 편집(신청 폼 항목에 예약 연결(장소·날짜·시간·연락처 등) 지정).
   - **[직접 신청]** 으로 관리자가 대신 접수(시작일~종료일, 종일 옵션 지원).
10. **오류 발생 시 해결 방법**: 승인 시 캘린더 반영 실패 → [대여가능공간]에서 **[연결 테스트]** 실행, 실패하면 Apps Script를 **최신 코드로, 액세스: 모든 사용자** 로 재배포. 이용결과 조회 안 됨 → 신청폼 연락처 항목에 예약 연결 '연락처(이용결과 조회용)' 지정.

## 12. 관리자 설정


1. **기능 설명**: 지출관리자 로그인 설정, 프로그램별 관리자 계정 생성·삭제·권한 부여, AI 초안 키 설정.
2. **관리자 권한 필요 여부**: **지출관리자 전용**(프로그램 관리자 접근 불가).
3. **접근 경로**: 좌측 **[관리자 설정]** (`/admin/admins`).
4. **처리 대상**: 관리자 계정·권한·키.
5. **단계별 처리 방법**
   1. **지출관리자 로그인** 카드에서 아이디·비밀번호를 설정(비밀번호는 변경 시에만 입력).
   2. **[관리자 추가]** 로 프로그램 관리자 계정 생성 → 아이디·비밀번호·이름 입력.
   3. **관리자 권한 부여** 에서 접근 허용할 메뉴를 체크(신청자 정보/신청폼 편집/공간대여 신청/사이트 설정).
   4. **담당 프로그램** 을 카테고리별로 토글합니다.
   5. **[저장]**.
   6. (AI) **AI 초안 작성 키** 에 Anthropic API 키·모델을 입력하고 **[저장]**(대리 신청 서술 초안 활성화).
6. **입력/수정 항목**: 지출관리자 아이디·비밀번호, 관리자 아이디·비밀번호·이름, 부여 메뉴, 담당 프로그램, AI 키·모델.
7. **처리 완료 후 확인사항**: 저장 성공. 각 관리자 로그인 시 부여된 메뉴만 노출.
8. **주의사항**: 관리자 설정 메뉴는 부여 대상이 아닙니다(지출관리자만). 아이디 중복 불가. 지출관리자 비밀번호가 없으면 로그인 자체가 막힙니다.
9. **실무 운영 팁**: 담당자에게 필요한 최소 메뉴만 부여하세요. 환경변수로 AI 키를 설정하면 화면 입력이 잠깁니다(`설정됨(환경변수)`).
10. **오류 발생 시 해결 방법**: `아이디가 중복됩니다` → 다른 아이디. `모든 관리자의 아이디를 입력해주세요` → 빈 계정 확인.

   - **푸터 설정**: 조직명·연락처 항목(유형·아이콘·텍스트·값) 추가/수정 → 상단 **[저장]**.
   - **사이드바 링크**: 라벨·링크(또는 파일 업로드)·아이콘·색상·작은 창 보기·카카오 스타일 설정 → 상단 **[저장]**.
   - **이용안내**: 섹션(제목+리치텍스트) 편집, **[사이드바에 이용안내 링크 추가]**, **[미리보기]**, **[기본 안내 불러오기]**, **[이용안내 저장]**.
   - **팝업 공지**: 팝업 사용·제목·내용·표시 기간 설정 → **[팝업 저장]**.
   - **파일 저장 경로**: 엑셀/지급신청서 파일명 형식·보관 경로(메모) 설정 → **[설정 저장]**.
6. **입력/수정 항목**: 각 탭 항목(위).
7. **처리 완료 후 확인사항**: 저장 후 홈/사이드바/이용안내에 즉시 반영.
8. **주의사항**: 파일 저장 경로는 파일명 형식·메모 용도이며 브라우저 특성상 자동 폴더 저장은 불가.
9. **실무 운영 팁**: 신청자 이용안내는 파일 재업로드 대신 **[이용안내]** 탭에서 편집하면 작은 창(/guide)으로 노출됩니다.
10. **오류 발생 시 해결 방법**: 저장 실패 시 재시도. 사이드바 파일이 안 열리면 업로드 형식(PDF·이미지) 확인.

## 14. 검토·지급 상태 단계 설정

1. **기능 설명**: 검토/지급 상태 라벨·색상·순서를 편집합니다.
2. **권한**: 관리자. 3. **접근 경로**: 신청 상세 → 상태 관리 카드 → **[옵션 편집]**.
4. **처리 대상**: 상태 옵션.
5. **단계별**: (1) **[옵션 편집]**. (2) 상태명·색상 수정, **[＋ 옵션 추가]**. (3) **[상태 옵션 저장]**.
6. **입력/수정 항목**: 상태명, 색상.
7. **완료 확인**: `상태 옵션이 저장되었습니다.` 모든 신청의 상태 드롭다운에 반영.
8. **주의**: 기본값(신청완료/검토중/보완요청/심의필요/승인/반려 등)을 임의 삭제 시 기존 데이터 표시에 주의.
9. **팁**: 색상으로 상태를 직관적으로 구분.
10. **오류 해결**: 저장 실패 시 재시도.

## 오류/장애 발생 시 일반 조치

- 캘린더 반영·웹훅 오류: [공간대여 → 연결 테스트], Apps Script 재배포.
- Drive 동기화 실패: `GOOGLE_SYNC_WEBHOOK_URL` 환경변수 확인.
- 로그인 불가: [관리자 설정]의 지출관리자 비밀번호 확인, `/api/admin/login-diag`.
- 데이터 저장 실패: 새로고침 후 재시도, Supabase 연결·app_config 확인.
- 상세는 [troubleshooting.md](./troubleshooting.md) 참고.
